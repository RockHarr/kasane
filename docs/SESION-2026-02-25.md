### 1. Configuración Inicial (30 min)

#### Problema Inicial
```bash
$ npm run dev
ERROR: Cannot apply unknown utility class 'gap-2'
```

**Causa raíz:** El proyecto tenía Tailwind CSS v4 instalado pero el código usaba sintaxis v3.

#### Solución Aplicada

**1.1 Instalación de dependencias**
```bash
npm install
npm install -D @tailwindcss/postcss
```

**1.2 Configuración de Vite**
```js
// vite.config.js
export default defineConfig({
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  }
})
```

**1.3 PostCSS Config para v4**
```js
// postcss.config.js
export default {
  plugins: {
    '@tailwindcss/postcss': {} // v4 usa plugin separado
  }
}
```

**1.4 Migración de style.css**
```css
/* ANTES (v3) */
@tailwind base;
@tailwind components;
@tailwind utilities;

/* DESPUÉS (v4) */
@import "tailwindcss";
@config "../tailwind.config.js";
```

**Resultado:** ✅ Servidor corriendo sin errores en `http://localhost:5173/`

### 2. Tailwind v4 — Fix de 13 Componentes Vue (45 min)

#### Problema
En Tailwind v4, cada `<style scoped>` es procesado independientemente por PostCSS, por lo que `@apply` no reconoce las utilidades sin `@reference`.

#### Solución
Agregado a **todos los componentes** con `@apply`:

```vue
<style scoped>
@reference "tailwindcss";
@config "../../../tailwind.config.js";

.my-class {
  @apply flex gap-2; /* Ahora funciona */
}
</style>
```

### 3. Tailwind Config — Tokens Faltantes (30 min)

#### Problema
Los componentes usaban clases custom que no existían en `tailwind.config.js`:
- `font-heading`, `font-body`, `font-mono`
- `bg-bg-elevated`, `text-text-muted`
- `accent-growth`, `accent-alert`, `accent-neutral`
- `shadow-glow-growth`, `shadow-glow-alert`

#### Solución
Actualizado `tailwind.config.js` con **12 tokens faltantes**:

```js
export default {
  theme: {
    extend: {
      colors: {
        bg: {
          primary: '#0a0e17',
          secondary: '#111827',
          elevated: '#1a2332', // ← NUEVO
        },
        text: {
          primary: '#f3f4f6',
          secondary: '#9ca3af',
          muted: '#6b7280', // ← NUEVO
        },
        accent: { // ← NUEVO (grupo completo)
          growth: '#00ffaa',
          'growth-bg': '#00ff88',
          alert: '#ffaa77',
          'alert-bg': '#ff6b35',
          neutral: '#5b9dff',
          'neutral-bg': '#3b82f6',

      },

        heading: ['Outfit', 'sans-serif'], // ← NUEVO

      },

        'glow-alert': '0 0 20px rgba(255, 107, 53, 0.3)',
```

### 6. Bug Fix — Overflow en Cards (15 min)

#### Problema Reportado
"Algunos valores quedan fuera de card"

**Causa:** Números largos con formato USD + `text-4xl` (36px) causaban overflow en cards estrechas.

#### Solución Aplicada

**6.1 MetricDisplay.vue — Texto Responsive**
```css
/* ANTES */
.metric-value {
  @apply font-mono text-4xl font-bold text-text-primary;
}

/* DESPUÉS */
.metric-value {
  @apply font-mono text-2xl sm:text-3xl lg:text-4xl font-bold text-text-primary;
  @apply break-words min-w-0;
}
```

**6.2 OCASimulator.vue — Grid Fix**
```css
.metrics-grid > * {
  min-width: 0; /* Permite que grid items se contraigan */
}
```

**6.3 BaseCard.vue — Overflow Hidden**
```css
.card {
  @apply rounded-xl transition-colors duration-200 overflow-hidden;
}
```

**Resultado:** ✅ Valores contenidos correctamente en cards, responsive en mobile/tablet/desktop
